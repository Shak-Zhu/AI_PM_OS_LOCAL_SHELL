## 范围管理原则

1. 范围来源：批准的 Requirements Register 与 Scope Baseline
2. WBS 使用 100% 规则，叶子工作包不超过 80 小时
3. Scope Baseline 未批准不得拆正式 WBS
4. 任何 P0 新增、删除或验收口径变化必须记录变更并由 Human Owner 批准
5. Baseline 变更必须更新相关 JSON 文件

## 范围变更流程

1. 识别：检测到范围变化需求
2. 影响分析：评估对时间、资源、质量和风险的影响
3. 记录：将变更请求写入 Change Log（PM_CHANGE_LOG.md）
4. 审批：由 Human Owner 或 Sponsor Approver 批准
5. 实施：更新相关文档和 JSON 数据
6. 通知：通知受影响的相关方
