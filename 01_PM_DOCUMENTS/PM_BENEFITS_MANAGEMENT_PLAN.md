# 效益管理计划

- 版本：v0.1
- 状态：Draft
- 最后更新：YYYY-MM-DD

## 目标效益

1. 降低项目文件、会议纪要、To-do 和报告的手工编制负担。
2. 提高范围、决策、变更和 Action 的可追溯性。
3. 通过 Memory Boot 降低上下文丢失风险。
4. 通过 Dashboard 缩短项目状态判断时间。

## 建议度量

- 初始化一个新项目所需人工时间。
- 从 transcript 到会议纪要/Pending Updates 的人工修订时间。
- 必填字段缺失率、未审批变更数、Markdown/JSON 不一致数。
- 用户在 30 秒内定位当前阶段、阻塞和下一步的成功率。

基线值和目标值将在 M1 前由 Human Owner 批准。
