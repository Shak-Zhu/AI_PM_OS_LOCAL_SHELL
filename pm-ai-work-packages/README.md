# Coder Work Packages

本目录用于当前壳实例初始化为真实项目后，存放该项目签发给 Coder 的工作包及其结果报告。

规则：

- 干净壳交付时只保留本说明文件，不包含 AI PM OS 产品开发期间的工作包。
- 新项目初始化和 Scope Baseline 批准前，不得签发正式 Coder Work Package。
- Scope Baseline 批准后，每份工作包必须包含唯一 WP ID、Required Project Files、scope_in、scope_out、Allowed Files、禁止事项、可验证验收标准、验证命令和 Read Evidence 要求。
- Coder 的交付状态使用 `implemented, pending PM/QC review`，不得代替 PM/QC 或 Human Owner 宣布验收完成。
- AI PM OS 产品自身的开发工作包属于 `_DEV_PROJECT_CONTROL/pm-ai-work-packages/`，不得复制到本目录。
